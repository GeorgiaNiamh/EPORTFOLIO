Coding task:

Question 1: The code lacked indentations that made the code error. I added indentations to the code such as for the if statement. 
									
									def factorial(n):
										"""Return factorial of n"""
										if n == 0:
											return 1
										else:
											return n * factorial(n - 1)

Question 2:
Python 2- In Python 2, raw_input() is used, but in Python 3, it should be replaced by input(). The if choice == "decode": block is incorrectly indented inside the if choice == "encode": block. It needs to be at the same level as the encode condition. When shifting the index for the letters in the encode and decode logic, it can exceed the bounds of the letters list. It should use modulus (%) to wrap around the list.
Question 3:
The output for the errors from Flake8 are very similar to those in pylint but Flake8 gives more detail and it is written in an easier to understand way. Use of input(): I replaced raw_input() with input() for Python 3 compatibility. Additionally, I converted the choice input to lowercase with .lower() to make it case-insensitive. Indentation of the decode block: I moved the decode block outside the encode block to ensure it only executes when the user selects "decode". Modulus (%) to prevent out-of-range index errors: Both in encoding and decoding, I used % len(letters) to make sure the index stays within the bounds of the letters string. Without this, indexing can go out of range when shifting letters. Printing the result: Changed print encoded to print(encoded) for Python 3 compatibility.
Question 4:
I think the code has a low cyclomatic complexity because it does not have too many branches and the only real contributor to the complexity is the if statement and this is only seen once making the complexity low.
