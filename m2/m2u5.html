Unit 5

The Cyclomatic Complexity is commonly considered in modules on testing the validity of code design today. However, in your opinion, should it be? Does it remain relevant today? Specific to the focus of this module, is it relevant in our quest to develop secure software? Justify all opinions which support your argument and share your responses with your team.
Cyclomatic complexity is a measure of how complex a piece of code is. Higher complexity means the code is harder to understand and test. While it's a useful tool, it doesn't tell the whole story about code quality.
In modern software development, cyclomatic complexity is still valuable. It can help identify complex parts of code that might be harder to maintain and test. It can also be used to guide code reviews and generate automated tests.
However, cyclomatic complexity has limitations. It doesn't always mean a complex code is bad, and other factors like code style and documentation are also important. Using tools and good practices can help manage complex code, even if it has high cyclomatic complexity.

References:


Shepperd, M. (1988). A Critique of Cyclomatic Complexity as a Software Metric. Software Engineering Journal. 3. 30 - 36. DOI: 10.1049/sej.1988.0003.
