Tower of Hanoi task:

What is the (theoretical) maximum number of disks that your program can move without generating an error?
What limits the number of iterations? What is the implication for application and system security?
Python has a limit on how many times a function can call itself. For the Towers of Hanoi, the number of recursive steps grows rapidly as the number of disks increases. By default, Python allows around 1000 recursive calls before it raises an error. This limits the number of disks to about 30â€“40 before the program crashes.
Each recursive call uses memory. If you try to solve the puzzle with too many disks, the program will eventually use up all available memory and crash.

UK Postcode task:

How do you ensure your solution is not subject to an evil regex attack?
Avoid overly complex or nested regex patterns. Our postcode regex is simple and doesn't contain catastrophic backtracking, which is often the cause of ReDoS attacks.
If your system allows it, set a maximum execution time for regex evaluations. This prevents the program from getting stuck processing malicious input.
Restrict the length of the user input before applying the regex. For postcodes, you can limit the input to something reasonable (e.g., 8-10 characters) since postcodes are short strings.
Before applying regex, do a basic check on input length or structure to filter out obviously invalid data, which avoids unnecessary regex checks.
